# الحصون الخمسة — رفيق الحفظ اليومي

تطبيق ويب (PWA) لتنفيذ برنامج **الحصون الخمسة** في حفظ القرآن الكريم للشيخ سعيد أبو العلا حمزة، بنفس ترتيب الحصون الأصلي:

١. حصن القراءة
٢. حصن التحضير
٣. حصن الحفظ الجديد
٤. حصن مراجعة القريب
٥. حصن مراجعة البعيد

التطبيق **ليس مصحفًا إلكترونيًا**، بل مساعد يومي لتسجيل الإنجاز في كل حصن، ومتابعة الالتزام، وإدارة جدول المراجعة.

## ملاحظة مهمة عن منهجية المراجعة

ترتيب الحصون الخمسة والتزامها اليومي مطابق للمنهج الأصلي. أما **جدول مراجعة البعيد التلقائي** (الفترات بين كل مراجعة) فهو خوارزمية تباعد تصاعدي قابلة للتعديل من الإعدادات (الفترة الأولى، معدل الزيادة، والحد الأقصى)، وليست منقولة حرفيًا من الكتاب لأنني لا أملك نصّ الجداول الدقيقة الموجودة في الكتاب الأصلي. يُنصح بمراجعة كتاب الشيخ وتعديل هذه القيم في الإعدادات لتطابق الجدول الوارد فيه بدقة إن رغبت في التطابق الحرفي.

## ماذا يوجد داخل المجلد

```
husun-app/
├── index.html      الصفحة الرئيسية
├── style.css        التنسيق
├── app.js           منطق التطبيق (كل البيانات تُحفظ محليًا على الجهاز)
├── manifest.json     ملف PWA (لتثبيت التطبيق كأيقونة على أندرويد)
├── sw.js             Service Worker للعمل دون إنترنت
└── icons/            أيقونات التطبيق
```

لا يحتاج التطبيق أي خادم خلفي (backend) أو قاعدة بيانات — كل البيانات (السجل اليومي، خطة المراجعة، الإعدادات) تُخزَّن في متصفح المستخدم نفسه عبر `localStorage`. لذلك يُنصح باستخدام ميزة **النسخ الاحتياطي** من الإعدادات دوريًا.

## النشر على Netlify (الأسهل)

1. اذهب إلى https://app.netlify.com
2. اسحب مجلد `husun-app` كاملًا وأفلته في صفحة "Deploys" (خاصية Drag & Drop)، أو اربط حساب GitHub بعد رفع المجلد إليه (الخطوات أدناه).
3. سيمنحك Netlify رابطًا مباشرًا مثل `https://your-app.netlify.app`.

## النشر على GitHub (عبر GitHub Pages)

1. أنشئ مستودعًا (repository) جديدًا على GitHub.
2. ارفع محتويات مجلد `husun-app` إلى المستودع (عبر الموقع مباشرة بسحب الملفات، أو عبر git):
   ```
   git init
   git add .
   git commit -m "الحصون الخمسة"
   git branch -M main
   git remote add origin <رابط-المستودع>
   git push -u origin main
   ```
3. من إعدادات المستودع Settings → Pages، اختر النشر من الفرع `main` والمجلد الجذر `/`.
4. سيصبح التطبيق متاحًا على رابط مثل `https://username.github.io/repo-name/`.

## تثبيته على أندرويد كتطبيق (بدون متجر)

بعد نشر الرابط:

1. افتح الرابط من متصفح Chrome على جهاز أندرويد.
2. اضغط على قائمة الثلاث نقاط (⋮) أعلى المتصفح.
3. اختر **"إضافة إلى الشاشة الرئيسية"** (Add to Home screen) أو **"تثبيت التطبيق"** (Install app).
4. ستظهر أيقونة التطبيق على الشاشة الرئيسية، ويعمل بواجهة مستقلة كأي تطبيق أندرويد عادي، ويعمل أيضًا دون إنترنت بعد أول فتح.

> **بخصوص التنبيهات:** يرسل التطبيق تنبيهات يومية أثناء تشغيله في المتصفح، لكن كونه تطبيق ويب (PWA) وليس تطبيقًا أصليًا مُثبّتًا من المتجر، فإن أندرويد قد يوقف التنبيهات في الخلفية عندما يكون التطبيق مغلقًا تمامًا لفترة طويلة، بحسب إعدادات توفير البطارية على الجهاز. للحصول على تنبيهات موثوقة تعمل دائمًا في الخلفية بنسبة ١٠٠٪، يلزم تحويل المشروع إلى تطبيق أندرويد أصلي (مثلًا عبر تغليفه بأداة Capacitor أو Trusted Web Activity) ونشره على متجر Google Play — وهي خطوة إضافية يمكن تنفيذها لاحقًا إن رغبت.

## التخصيص من داخل التطبيق

من شاشة **الإعدادات** يمكن ضبط:
- مقدار الحفظ والقراءة اليوميين.
- وقت تنبيه كل حصن على حدة، وتفعيله أو إيقافه.
- الوضع الليلي/النهاري وحجم الخط.
- تنزيل نسخة احتياطية (JSON) واستعادتها على جهاز آخر.
